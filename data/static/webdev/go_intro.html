What is Go?

Go is an open source programming language aimed at building simple, reliable, and efficient software.

Gopher image by Renee French, licensed under Creative Commons 3.0 Attributions license.

Go is a general purpose language designed at Google by Robert Griesmer, Rob Pike, and Ken Thompson. The language is statically typed, explicit, compiled, and syntactically similar to C, with the added benefits of garbage collection and structural typing.

History

Go was designed in 2007 with the intention of improving productivity at Google. Go was first made publicly available in March 2012. Since such, it has become the fastest growing language on GitHub, as well as the highest paid programming language in 2018. Since Goâ€™s release, the language has seen many iterations, and as of March 2019 the current release is Go 1.12, the twelfth public release.

Rob Pike, best known for his work on Go and at Bell Labs, was also a member of the Unix team and was involved in the creation of the Plan 9 operating system. With Ken Thompson (another co-author of Go), he created the UTF-8 character encoding definition.

Ken Thompson is considered a pioneer of computer science. Having worked at Bell Labs for most of his career, Thompson designed and implemented the original Unix operating system.

Go is widely used in production at Google as well as many large organisations such as Apple, IBM, Dropbox, Netflix, BBC, Uber, Medium, Docker, SoundCloud, Cloudflare, Kubernetes, and many more.

Design

The aim of the language was to solve problems specifically within Google, in an era of multicore, networked machines and extremely large codebases. The designers of Go wanted to address criticisms of other languages used at Google such as Python and C, while retaining their useful characteristics. This means Go is particularly suited to development on the web, processing huge volumes of data efficiently and with high performance.

Go adopts characteristics from C++ and Java, such as static typing and run-time efficiency, as well characteristics from Python and JavaScript, such as readability and usability.

Syntax

Go modifies the syntax of C in an attempt to keep code concise and readable. Efforts have been made to make Go code minimal, as to reduce the potential risk of errors. Let's analyse some syntax features of a basic if statement in Go.

if myInt > 5 {   doSomething() }

The first notable feature is that there are no brackets around the conditional, contrasting the syntax of C. Secondly, the opening brace is written on the same line. In other languages, the programmer may choose to write the opening brace on the next line. Part of the appeal of Go is that all code is uniform and follows the same structural form. To achieve this, the Go formatter will ensure the opening brace remains on the same line, as in the example above. This makes working in a team or with others' code easier, as you know what to expect when reading it.

Additionally, note the lack of statement terminator. Semi-colons can be used to terminate statements in Go, however the Go formatter will remove semi-colons in an effort to keep the code as uncluttered and as readable as possible.

Similarly, round brackets can be written around the if statement conditional, however this too is removed by the Go formatter.

With these small differences, code remains uniform, easy to read, and easy to write.

Go Commands

From the Go Docs:

Usage:

go <command> [arguments]

The commands are:

bug: start a bug report
build: compile packages and dependencies
clean: remove object files and cached files
doc: show documentation for package or symbol
env: print Go environment information
fix: update packages to use new APIs
fmt: gofmt (reformat) package sources
generate: generate Go files by processing source
get: download and install packages and dependencies
install: compile and install packages and dependencies
list: list packages or modules
mod: module maintenance
run: compile and run Go program
test: test packages
tool: run specified go tool
version: print Go version
vet: report likely mistakes in packages

Why Go?

Go is embraced by many impactful and powerful organisations including Google, Apple, IBM, and those mentioned above.

Go has solid credentials. It is financed by Google, and designed by pioneers in the computer science field.

Go is open source.

Go is fast, on a number of levels. Writing, compiling, and executing Go is designed to be quick and efficient.

Go has fantastic support for concurrency, through the use of goroutines.

Go is garbage collected. This has a significant improvement on performance and makes concurrency more efficient.

Go has useful and helpful tools, which make development in Go fluid, and makes working with other Go code easier.
